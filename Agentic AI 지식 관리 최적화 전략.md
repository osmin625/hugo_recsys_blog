# **AGENTS.md: Agentic AI 기반 로컬 퍼스트 지식 관리 및 퍼블리싱 아키텍처 명세서**

본 문서는 AI 코딩 에이전트(Cursor, Claude Desktop, Gemini CLI 등)가 사용자의 지식 관리(PKM) 시스템을 마이그레이션하고 무인 자동화 파이프라인을 구축하기 위해 읽고 실행해야 하는 프로젝트 명세서입니다. 이 문서는 수많은 오픈소스 프로젝트에서 에이전트의 작업 지시 표준으로 채택하고 있는 AGENTS.md 및 CLAUDE.md 형식을 차용하여 작성되었으며, 에이전트가 시스템을 완벽히 이해할 수 있도록 프로젝트 컨텍스트, 아키텍처 결정 사항, 코딩 규칙, 구현 과제 등 4가지 핵심 섹션으로 구성되어 있습니다.

## **1\. Project Context (프로젝트 개요 및 비전)**

* **핵심 목표:** 클라우드 플랫폼 종속성과 구독 비용을 0원으로 수렴시키는 '로컬 퍼스트(Local-First)' 개인 지식 관리 시스템을 구축하고, 축적된 데이터를 외부 플랫폼(블로그, 유튜브 등)으로 자동 배포하는 엔드투엔드 파이프라인을 구현합니다.  
* **에이전트의 역할:** 이 명세서를 바탕으로 고수준의 비전을 구체적인 코드와 폴더 구조, 자동화 스크립트로 확장하고 구현하는 것입니다.  
* **운영 환경:** Windows, macOS, Linux, iOS, Android 등 다중 OS 환경 오프라인 연동.

## **2\. Architecture Decisions (아키텍처 결정 사항)**

기존 시스템을 개편할 때 에이전트가 반드시 준수하고 구성해야 하는 기술 스택과 구조적 결정 사항입니다.

## **2.1. 데이터 포맷 및 인터페이스 설계**

* **기본 포맷:** 지식 본문은 순수 마크다운(Markdown)으로 작성하여 에이전트의 파싱 효율을 극대화합니다. 복잡한 XML이나 JSON 사용은 본문 작성에 지양합니다.  
* **메타데이터 분리:** 시스템 설정 및 문서 메타데이터(제목, 태그 등)는 파일 최상단의 YAML 프론트매터(Front Matter)에 고립시킵니다.  
* **Agent-Flavored Markdown (AFM) 도입:** 에이전트의 지시사항(자연어)과 구동 설정(YAML)을 단일 파일 내에 분리하여 정의함으로써 특정 프레임워크에 종속되지 않는 이식성을 확보합니다.

## **2.2. 디렉토리 및 노드 아키텍처**

* **Flat 디렉토리 구조:** 깊게 중첩된 폴더 구조를 평면화(Flatten)합니다.  
* **접두어(Prefix) 명명 규칙:** 폴더를 탐색하지 않아도 노드의 성격을 즉시 알 수 있도록 파일명에 강력한 메타데이터를 부여합니다 (예: Log \- 1-1 \- Manager, Course \- Agent Skills).  
* **단일 진실 공급원:** 개별 노트는 파편화되지 않고 특정 주제에 대해 자립적이고 완결된 답변(Complete, self-contained answers) 형태를 갖추어야 합니다.

## **2.3. 무비용(Zero-Cost) 로컬 퍼스트 동기화 망**

* **실시간 파일 동기화:** P2P 프로토콜인 싱크띵(Syncthing)을 통해 E2E 암호화 기반의 로컬 백그라운드 동기화를 구현합니다 (서버/구독료 배제).  
* **iOS 샌드박스 우회:** iOS 환경에서는 SyncTrain 등의 앱과 Apple 단축어(Shortcuts)의 재귀적 루프(15회 반복, 10초 대기)를 결합하여 백그라운드 동기화를 강제 유지하는 스크립트를 구성합니다.  
* **버전 관리의 분리:** Git은 실시간 동기화 용도로 사용하지 않으며, 오직 유휴 상태(Idle State)에서의 스냅샷 백업 용도로만 제한합니다.

## **2.4. 경량화된 AI 추론 및 RAG 인프라**

* **로컬 벡터 DB:** 무거운 컨테이너 기반 DB(ChromaDB, Pinecone 등) 대신, 단일 파일 기반의 SQLite와 sqlite-vec 확장 모듈을 결합하여 이진 양자화(Binary Quantization) 기반의 초고속 검색을 구현합니다.  
* **그래프 추론 (Graph RAG):** LightRAG 엔진을 로컬로 구동하여 마크다운 노드 간의 엔티티와 논리적 관계를 추출, 메타인지적 하이브리드 검색 환경을 구축합니다.  
* **Model Context Protocol (MCP):** 특정 노트 앱의 플러그인에 종속되지 않도록, 로컬 디렉토리를 MCP 서버로 노출하여 Claude Desktop, Cursor 등의 외부 에이전트가 파일 시스템에 안전하게 접근하고 CRUD 작업을 수행하게 합니다.

## **2.5. 퍼블리싱 자동화 파이프라인**

* **iPaaS 엔진:** Zapier나 Make.com 대신, Docker로 셀프 호스팅된 n8n(또는 Activepieces)을 사용하여 자동화 워크플로우를 무료로 오케스트레이션합니다.  
* **정적 블로그 확장 (SSG):** 로컬 저장소의 특정 폴더 상태가 변경되면 Python 전처리 스크립트가 위키 링크를 표준 링크로 변환하고, GitHub Actions가 Hugo를 트리거하여 Netlify/GitHub Pages에 블로그를 무인 배포합니다.  
* **유튜브 비디오 확장:** n8n과 연동된 에이전트가 문서를 스크립트로 파싱하고, Python 스크립트가 TTS(음성 합성), yt\_dlp(푸티지 수집), MoviePy(자동 편집)를 거쳐 YouTube Data API로 자동 업로드하는 파이프라인을 구축합니다.

## **3\. Coding Conventions & Boundaries (코딩 규칙 및 제약 사항)**

에이전트가 코드를 작성하거나 스크립트를 생성할 때 절대로 위반해서는 안 되는 경계 사항입니다.

* **Zero-Ops 원칙:** 시스템 인프라 구축 시, 데이터베이스 유지를 위해 별도의 데몬이나 복잡한 Docker 오케스트레이션을 요구하지 마십시오. SQLite 기반의 단일 파일 접근 방식을 최우선으로 고려해야 합니다.  
* **크기 제약 (10MB 분할 규칙):** LLM 컨텍스트 윈도우의 효율성을 위해, 단일 마크다운 파일의 크기가 10MB(약 200페이지 분량)를 초과할 경우 논리적 단위(Part 1, Part 2 등)로 분할 처리하는 전처리 로직을 반드시 포함하십시오.  
* **종속성 회피:** 특정 AI 공급자(OpenAI, Anthropic 등)의 API에 락인(Lock-in)되는 하드코딩을 피하고, 환경 변수(.env) 기반으로 쉽게 LLM 공급자를 교체할 수 있도록 모듈화하십시오.  
* **Git 무결성 보호:** Syncthing과 Git 워크플로우가 충돌하여 저장소가 손상되지 않도록, Git 커밋 자동화 스크립트 실행 전 파일 잠금(Lock) 상태를 확인하는 방어적 코드를 작성하십시오.

## **4\. Implementation Tasks (구현 과제)**

에이전트는 다음 작업들을 순차적으로 분석하고 구현 계획을 수립한 뒤 코드를 생성해야 합니다.

1. **디렉토리 정규화 모듈 개발:** 기존 계층형 폴더 구조를 Flat 형태로 마이그레이션하고, 모든 파일에 접두어 명명 규칙을 적용하는 Python 스크립트 작성.  
2. **MCP 서버 연동 스크립트 구성:** 지정된 지식 저장소 디렉토리를 루트로 삼아 읽기/쓰기 권한을 제어하는 obsidian-mcp-server 구성 파일(JSON) 생성.  
3. **SQLite-vec RAG 파이프라인 구축:** 마크다운 파일들을 청킹(Chunking)하고, 임베딩을 생성하여 로컬 .sqlite 파일 내에 저장 및 업데이트하는 데이터 파이프라인 스크립트 작성.  
4. **Hugo 퍼블리싱 자동화 작성:** 옵시디언 고유의 링크(\[\[ \]\])를 마크다운 표준 링크로 치환하고 에셋 폴더를 복사하는 전처리 스크립트 작성 및 GitHub Actions 배포용 .yml 스크립트 생성.  
5. **비디오 렌더링 파이프라인 구성:** 마크다운 텍스트를 입력받아 TTS 오디오를 생성하고 MoviePy를 통해 비디오로 병합하는 무인화 Python 스크립트 템플릿 작성.
